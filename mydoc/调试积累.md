在`xla/examples/axpy/stablehlo_gpu_compiler.cc`中获取GPU的PJRT：
```c++
absl::StatusOr<std::unique_ptr<PjRtClient>> CreateGpuClient() {
  xla::GpuClientOptions options;
  // 可按需调整 options，例如选择特定设备集合：
  // options.allowed_devices = std::set<int>({0});
  return xla::GetXlaPjrtGpuClient(options);
}
```
可能会有错误：
```
Failed to create GPU client: NOT_FOUND: Could not find registered platform with name: "cuda". Available platform names are: 
```
此问题是因为二进制运行时没有将 CUDA StreamExecutor 平台（以及/或 PJRT GPU 插件）的静态注册器链接进来，因此`xla/stream_executor/cuda/cuda_platform.cc`中相应的静态初始化函数没有运行，PlatformManager::RegisterPlatform(...) 从未被调用。
解决：把实现 CUDA 平台注册的 object file / 库链接到你的二进制，使得文件里的模块初始化语句（例如 STREAM_EXECUTOR_REGISTER_MODULE_INITIALIZER 或 REGISTER_PJRT_PLUGIN 的静态注册宏）在进程启动时执行。解决办法是在该 cc_binary 的 deps 中添加对应的 Bazel target： //xla/stream_executor:cuda_platform（会包含 cuda_platform.cc 的静态初始化）和 //xla/pjrt/plugin/xla_gpu:gpu_static_registration（注册 PJRT GPU 插件）。
因此如果后续要增加GPU的PjRT，也需要参考`xla/stream_executor/cuda/cuda_platform.cc`来实现相应的平台静态注册。
